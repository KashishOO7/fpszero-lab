# The Lab

> Theory without experiment is philosophy. This is where theory becomes code, and code becomes understanding.

---

## What Happens Here

The Research section documents active experimentation — real projects, real code, real results. This is not cleaned-up tutorial content. It is a **working lab notebook**.

Format for each project:

1. **Hypothesis** — What I'm trying to understand or build
2. **Approach** — Method, tools, architecture
3. **Results** — What actually happened (including failures)
4. **Synthesis** — What it changed about my mental model

## Lab Rules

1. **Document everything** — including dead ends. A failed experiment is data.
2. **No copy-paste without understanding** — if I can't explain every line, I rewrite it.
3. **Reproduce before you publish** — results are verified before going on the page.
4. **Flag speculative content** — clearly marked with `!!! note "Hypothesis"` blocks.
